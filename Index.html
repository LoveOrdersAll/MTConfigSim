Deployment Context: This guide ensures your Linux system interfaces correctly with the node data streams over USB serial connections or network-mapped brokers.

1. Apply Node Configuration via Python CLI

Ensure the node properties match your simulation settings over local USB:

pip install meshtastic
meshtastic --set owner.name "Node" --set device.role CLIENT

2. Fix Hardware Serial TTY Permissions

If interacting via serial terminal on Linux, grant access to the dialout group:

sudo usermod -a -G dialout $USER
# Log out and log back in for changes to apply

3. Live MQTT Stream Inspection (Mosquitto)

To monitor live node telemetry and payloads passing through your gateway broker:

# Subscribe to all JSON/Protobuf packets under the regional root mesh topic
mosquitto_sub -h localhost -t "msh/US/2/c/#" -v

4. Automated Node Data Logger Service

Create a persistent systemd daemon loop to record background JSON traffic:

[Unit]
Description=Meshtastic MQTT Logger
After=network.target

[Service]
ExecStart=/usr/bin/python3 -m meshtastic.mqtt_logger --host localhost --topic msh/US/2/c
Restart=always
User=root

[Install]
WantedBy=multi-user.target
